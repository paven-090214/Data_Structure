## 키보드 입력시 Input, Output처리 과정

1. 키보드에서 입력 발생한다.(Input)  
2. 인터럽트(Interrupt) 발생한다. → OS에 알린다.
3. OS가 입력 데이터를 RAM(메모리)에 미리 할당된 입력 버퍼(I/O Buffer)에 데이터를 순서대로 기록한다.(Write)  
4. CPU가 메모리에서 데이터를 읽어 처리한다. (연산)  
5. OS가 출력 시스템에 전달한다.
   - GUI (그래픽 화면)
   - 콘솔 (터미널)
6. 화면에 출력한다. (Output)

컴퓨터는 크게 소프트웨어(S/W)와 하드웨어(H/W)로 구분한다.
CPU의 동장 모드는 User mode와 Kernel mode로 나뉜다.
User mode에서는 VS Code, 크롬, 게임등의 프로그램이 실행된다.
Kernel mode에서는 OS가 실행되며, 하드웨어 접근과 같은 권한이 필요한 작업을 수행한다.

Hello World라는 문자열 출력 프로그램(프로세스)를 실행

kernel mode 내부에 H/W로 갈 수 있도록 만드는 추상화된 인터페이스(파일의 형태를 띔)를 제공함

1. Hello World 문자열을 출력하는 프로그램을 실행하면 프로세스가 생성이 된다.
2. 이 프로세서는 User mode에서 실행이 된고, Wirte한다.
3. 이 과정에서 kernel mode로 전환후, OS가 H/W에 접근을 수행한다.
4. 운영체제는 콘솔과 같은 장치를 파일처럼 사용할 수 있는
추상화된 인터페이스를 제공하며,
Windows에서는 이를 CON과 같은 장치 파일로 표현한다.
5. 프로그램이 write한 데이터는 커널을 통해 콘솔 장치로 전달되며,
이 과정에서 버퍼에 저장되었다가 화면에 출력된다.





EX). 만약 키보드의 입력값을 input하는 프로그램인 경우
1. 키보드의 입력을 담당하는 디바이스 존재
2. 그것의 운영채제 요소 존재
3. 것을 인터페이스 추상화(콘솔 인풋)
4. 그 값을 READ함(누가 하는지 알려줘  CPU가 하는건가?)
5. INPUT하는 프로세스를 실행 (READ)
6. 그 wirte한게 kernel mode에 콘솔로 갈 수 있는 존재에 들어간다. ->  이 공간에 버퍼가 존재해서 Write할때마다 저장된다
그래서 Hello World의 문자열이 콘솔에 찍힌다.
----
키보드 입력이 발생하면 인터럽트가 발생하고,
CPU는 Kernel mode로 전환된다.

운영체제는 키보드 입력을 받아 메모리(입력 버퍼)에 저장한다.

이후 사용자 프로그램이 read() 시스템콜을 호출하면,
운영체제가 버퍼에 저장된 데이터를 프로그램에 전달한다.

이 과정에서 CPU는 Kernel mode와 User mode를 오가며 동작한다.
